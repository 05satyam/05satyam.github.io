About Me

As a tech guy somehow we all encountered the word annotation in tech careers, so what is an annotation? An annotation is a form of syntactic metadata and used to provide additional information Examples of few built in annotation : @Override, @Deprecated, @SuppressWarnings etc.

First question which comes to mind is , How does Java know which is a custom annotation? This

@interface

keyword tells Java this is a custom built annotation. Eg. @interface MyCustomAnnotation{} - Here, MyCustomAnnotation is the custom annotation name.

Creating an annotation most importantly requires two tags:

(1) a retention policy (2) a target

A

retention policy

specifies how long (i.e. program lifecycle) the annotation should be retained for. For example, annotations may be retained during compile-time or runtime depending on the retention policy associated with that particular annotation.

Retention policies available:

RetentionPolicy.SOURCE, RetentionPolicy.CLASS, RetentionPolicy.RUNTIME

The target tag of an annotation tells which Java constructs an annotation can be applied to. For example, some annotations may be valid for methods only, while others may be used for both classes and fields. One or more targets may be associated with an annotation

Target policies available are:

ElementType.TYPE, ElementType.FIELD, ElementType.METHOD, ElementType.CONSTRUCTOR, ElementType.LOCAL_VARIABLE, ElementType.ANNOTATION_TYPE, ElementType.PARAMETER

Annotation Parameters

Annotations can also be associated with parameters. These parameters may be a primitive (such as int),annotation, enum, String, class, or an array of any of the five preceding types. Types of Annotation on the basis of parameters passed to custom annotation.

1.Marker Annotation –

This type of annotation only used to mark a declaration. These types of annotation do not contain any method and no data. Here, what matters is its presence. Eg: @MarkerAnnotation() ---> custom annotation @Override ---> java inbuilt marker annotation

2.Single-Value Annotation --

As the name suggest, these annotation allow only one member, hence an annotation that has one method is known as Single-Value Annotation. We only need to specify the value of that member when annotation is used. Eg: @SingleValueAnnotation(“annotate”)

3.Multi-Value Annotation-

As the name suggests, an annotation that consists of multiple data members, values. Eg: @MultiValueAnnotation(name=“multivalue”, value=”category”)

//Sample Example to create custom annotation import java.lang.annotation.*; import java.lang.reflect.*; //Single-Value Annotation @Retention(RetentionPolicy.RUNTIME) @Target(ElementType.METHOD) @interface MyCustomAnnotation{ int value(); } //Applying annotation class MyCustomAnnotation1{ @MyCustomAnnotation(value=1) public void print(){ System.out.println("hello"); } public static void main(String args[])throws Exception{ MyCustomAnnotation1 customAnnotation1=new MyCustomAnnotation1(); Method m=customAnnotation1.getClass().getMethod("print"); MyCustomAnnotation customAnnotation=m.getAnnotation(MyCustomAnnotation.class); System.out.println("value : "+customAnnotation.value()); } } Output: value : 1
